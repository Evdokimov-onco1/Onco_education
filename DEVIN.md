# Задание для Devin — репозиторий Onco_education

Контекст: это рабочая среда кафедры онкологии для подготовки ФОС к аккредитации. Стек намеренно простой: статический сайт на GitHub Pages + `fos-data.json` как общее состояние + Google-документы для содержания. Никакого backend и никакого Notion.

## Роль Devin в инфраструктуре
Devin — агент сопровождения репозитория. Он деплоит сайт, поддерживает `fos-data.json` в актуальном виде и по запросу генерирует заготовки содержания ФОС. Каждая задача — отдельный Pull Request в `main`.

## Задача 1 — развернуть сайт (сделать первым)
1. Убедиться, что в корне есть `index.html`, `fos-data.json`, `README.md`.
2. Включить GitHub Pages: branch `main`, папка `/(root)`.
3. Проверить, что `https://evdokimov-onco1.github.io/Onco_education/` открывается и что дашборд подгружает `fos-data.json` (в ячейках видны ссылки на Google Drive, распределение по темам совпадает с файлом).
4. В PR приложить рабочую ссылку.

## Задача 2 — держать данные в синхроне
Когда приходит обновлённый `fos-data.json` (координатор экспортирует из дашборда): принять файл, заменить в корне, открыть PR с понятным заголовком (например «update: приёмка ФОС Онкология 2 сем»). Схема файла: ключ `дисциплина:семестр` → `{ owners: [по темам], approved: bool, drive: url }`. Не менять число элементов в `owners` (оно равно числу тем ФОС).

## Задача 3 — заготовки трёх недостающих ФОС
Для ФОС со статусом «создать с нуля» (Химиотерапия 1 сем, Хирургия 2 сем, Лучевая терапия 2 сем) сгенерировать черновики тестовых заданий и вопросов по темам из соответствующего Google-документа, строго по норме: 80 тестов (закрытые а/б/в/г + открытые), ≥30 теоретических вопросов, 15 клинических задач с ключами, таблица соответствия компетенциям. Результат — не в репозиторий, а текстом в PR-описании для переноса врачами в Google-документ. Медицинское содержание обязательно вычитывает профильный врач; Devin готовит только структурную заготовку.

## Чего не делать
- Не подключать сторонние сервисы, базы данных, сборщики — сайт должен оставаться одним статическим `index.html`.
- Не трогать медицинское содержание без проверки профильным врачом.
- Не переписывать `fos-data.json` по своему усмотрению — только принимать присланную версию.
