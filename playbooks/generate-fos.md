# Playbook: сгенерировать недостающий ФОС

## Назначение
Написать ФОС для дисциплины, у которой его нет (`status: "missing"` в `fos-data.json`), так, чтобы он был **структурно неотличим** от пяти существующих ФОС в папке `fos/`.

Применяется к: `him1` (Химиотерапия 1 сем), `hir2` (Хирургия 2 сем), `lt2` (Лучевая терапия 2 сем).

## Перед началом
1. Прочитать эталон: `fos/onk2-onkologiya-2-sem.md` — это образец нужной структуры.
2. Взять темы и ответственных из `fos-data.json` → `fos.<ключ>.themes`.
3. Тематический план (часы, формы аттестации) взять из файла `31.08.57 Онкология ТП нов.xlsx` — его предоставит координатор, если его нет в репозитории.

## Обязательная структура документа (порядок строгий)

### 1. Шапка
```
<Код> <Название дисциплины>

Тематический план <N> семестр
```
Затем таблица тематического плана: `№п/п | Наименование темы | ВСЕГО контактной работы | Лекции | Семинары | Практические занятия | Консультации | Самостоятельная работа | ВСЕГО часов`, строка `Всего за N семестр` с итогами.

### 2. Тесты — ровно 80 заданий

**Задания 1–20 — закрытые.** Заголовок: `Тесты:` затем `Прочитайте вопрос и выберите правильный ответ:`

- **1–5** — компетенция **ПК-1** (профилактика, раннее выявление, консультирование)
- **6–10** — **ПК-6** (диагностика и лечение)
- **11–14** — **ПК-8** (реабилитация, восстановительное лечение)
- **15–17** — `Установите соответствие между ...` (варианты а/б/в/г ↔ 1/2/3/4)
- **18–20** — `Установите правильную последовательность ...`

Формат каждого задания:
```
1. <Клиническая ситуация или вопрос>

а) <вариант>

б) <вариант>

в) <вариант>

г) <вариант>

Ответ: 
```
Поле `Ответ:` оставляется пустым — ответы идут отдельным блоком (так в оригиналах).

Затем:
```
Ключи:

1. б

2. в
...
15. а3, б4, в1, г2      ← для заданий на соответствие
18. бваг                ← для заданий на последовательность
```

**Задания 21–80 — открытые.** Заголовок: `Прочитайте вопрос и запишите краткий ответ:`

Блоки строго по компетенциям:
- **21–40** — ПК-1 (профилактика, скрининг, факторы риска, маршрутизация)
- **41–60** — ПК-6 (диагностика, лечение, методы)
- **61–80** — ПК-8 (реабилитация, поддерживающая терапия, качество жизни)

Формат:
```
21. <Вопрос, требующий краткого ответа одним термином>

Ответ: 
```
Затем блок `Ключи:` с короткими ответами (`21. Маммография`, `22. Менее 1 мм` и т.п.).

Затем:
```
Соотношение заданий и компетенций

ПК-1: задания № ...
ПК-6: задания № ...
ПК-8: задания № ...
```
(перечислить номера всех 80 заданий по компетенциям)

### 3. Теоретические вопросы — ровно 30

```
Теоретические вопросы

Соотношение вопросов и компетенций

ПК-1: вопросы № 1–10
ПК-6: вопросы № 11–20
ПК-8: вопросы № 21–30

Вопросы

1. <Формулировка>
...
30. <Формулировка>
```
Стиль формулировок — как в оригиналах: либо назывной (`Рак пищевода. Клиническая картина. Классификация. Диагностика.`), либо развёрнутый (`Охарактеризуйте принципы...`, `Раскройте значение...`, `Объясните, каким образом...`).

### 4. Клинические задачи — ровно 15

Каждая начинается заголовком, где **явно указана тема из тематического плана**:
```
**Задача 1. <Полное название темы из ТП>**

<Клинический случай: возраст, пол, жалобы, анамнез, данные обследования,
 осложняющее обстоятельство — отказ пациента, коморбидность, соцфактор.
 Объём 120–200 слов.>

Вопросы:
1. <...>
2. <...>
3. <...>

Эталон ответа:
<Развёрнутый разбор>
```
Распределение задач по темам — равномерно, как задано в `fos-data.json` (поле `tasks` каждой темы).

Затем:
```
Соотношение задач и компетенций

ПК-1: задачи № 1–5
ПК-6: задачи № 6–10
ПК-8: задачи № 11–15
```

---

## Критерии готовности
- [ ] Ровно 80 тестов, 30 теоретических вопросов, 15 задач
- [ ] Все три блока «Соотношение ... и компетенций» присутствуют и покрывают все номера
- [ ] Ключи есть ко всем 80 тестам
- [ ] Каждая задача подписана темой из тематического плана
- [ ] Номера заданий по темам совпадают с квотами в `fos-data.json`
- [ ] Файл сохранён как `fos/<ключ>-<дисциплина>-<N>-sem.md`
- [ ] В `fos-data.json` у этого ФОС: `status` → `"exists"`, `totals` заполнены, `plan` у тем убран

## Ограничения (обязательно)
- **Медицинское содержание проверяет профильный врач.** В описании PR указать: «Черновик. Требуется проверка: <фамилия ответственного из fos-data.json>».
- Не выдумывать дозы, схемы терапии и цифры, если не уверен — формулировать вопрос так, чтобы ответ был терминологическим, а не числовым.
- Опираться на действующие клинические рекомендации РФ и материалы существующих ФОС.
- Не менять структуру и нумерацию существующих ФОС.
